    <h1 id="class-LogWalker" class="class">
  class LogWalker
</h1>

<section class="description">

    <p>The class LogWalker abstracts the most common access procedures to visit the log files.</p>

    <p>The class is tailored over psmetric01 case where a single directory contains many machines logs.</p>

    </section>

<div id="method-i-rot1Files" class="method-detail ">

  <div class="method-heading">
    <span class="method-name">rot1Files</pan><span
      class="method-args">() { |tmp| ... }</span>

    <span class="method-click-advice">click to toggle source</pan>

  </div>

  <div class="method-description">

    <p>Return <code>Enumerator</code>. Return all log files that have been rotated

    exactly one time. There files in out setup are all the one that have a date but are not compressed.</p>

    <p>It is useful to be able to spot these files because when a log file is empty</p>

    <p>we will look at the last logline in the rot1 file. The rot1 file can not be empty by construction. Indeed, according to our current logrotate.conf in psmetric01, files are rotated only if they contain at least 100k, therefore the rotated file can not be empty.</p>

    <pre>sto.rot1Files

    &gt; #&lt;Enumerator: …&gt;¶ ↑

    sto.rot1Files.first(3)

    &gt; [&quot;/mnt/lcls-logs/psana101/messages-20181028&quot;,¶ ↑

    &quot;/mnt/lcls-logs/psana101/secure-20181028&quot;]</pre>
