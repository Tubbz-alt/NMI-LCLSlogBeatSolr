    <h1 id="class-LogLine" class="class">
  class LogLine
</h1>

<section class="description">

    <p>The class <a href=“LogLine.html”>LogLine</a> is specifically to represent logfile lines as seen in psmetric01. Here a few examples of log lines.</p>

    <pre>Oct 31 11:06:01 psana101 systemd: Stopping User Slice of nmingott. Oct 31 14:22:43 psana101 sshd: Connection closed by 172.21.33.16 port 54600 [preauth]</pre>

    <p>We observe there are 4 elements of interest: the <strong>time</strong> the log arrived, the <strong>hostname</strong> of the machine who created the log, the <strong>application</strong> who created the log and finally the <strong>message</strong>. This structure dependes on the servers configuration.</p>

    <p>We streee an important point, in the date there miss the year. For this reason we implement a few methods to return a date including the year.</p>

    </section>

  <div class="method-description">

    <p>Parse the <a href="LogLine.html">LogLine</a> string and returns a Ruby

    Hash. Return Hash.</p>

    <p>These keys are added to the onse found parsin the line parseFail : in general is false. If there were error parsing the line its value is set to true and the whole line content is set under the key message.</p>

    <p>The <strong>dates</strong> are converted to a format Elasticsearch can recognize. ie. “2018-11-10T22:48:01”. When there is an error in parsing the date is set at the moment of paring the data.</p>
