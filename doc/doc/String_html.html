    <h1 id="class-String" class="class">
  class String
</h1>

<section class="description">

    <p>We overload the class String with some methods to simplify the management of dates.</p>

    <p>Dates appearing in LCLS logfiles have this format: <strong>Oct 21 10:23:11</strong> . We call such formatted strings LogDate. We do not definie a specific class for this kind of objects. The important thing here is that LogDate do not contain the year.</p>

    <p>We say that a <a href=“String.html”>String</a> in this format: <strong>2018 Oct 21 10:23:11</strong>, is a string in the FullLogDate format.</p>

    <p>The following methods permit to move fast between these kind of objects: LogDate, FullLogDate and DateTime.</p>

    <p>When a conversion is needed between LogDate and some other kind of date representation requiring the <strong>year</strong>, the the <strong>year is guessed</strong> with this rule. “We select the year such that appending it before the LogDate string, we get a valid FullLogDate string representing the nearest past date respect to present”.</p>

    <p><strong>See also</strong>. companion methods overloading the DateTime class.</p>

    </section>
